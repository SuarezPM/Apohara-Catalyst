# Webhook deferral — v1.0 → v1.1

The `/webhook` endpoint in `packages/github-bridge/src/webhook.ts` returns HTTP 501 in v1.0. Full push-event delivery is deferred to v1.1.

## Why deferred

Push events require a two-track delivery worker (SSE for in-process orchestration + a separate webhook delivery worker for external consumers). The v1.0 acceptance gate (poll-only github-bridge) does not need either; we ship the smaller surface area and harden it before expanding.

## v1.1 plan

- Implement a webhook delivery worker that:
  - Verifies `x-hub-signature-256` against the secret from `apohara.secrets`.
  - Drops events older than 5 minutes (replay protection).
  - Enqueues into the orchestration DB on the same `issue-{N}` thread the poller already uses.
- Add an SSE endpoint for the desktop UI to receive real-time push events.
- Migrate `webhook.ts` from 501 → real handler.

## Tests pinning the 501 contract

`tests/integration/github-bridge/webhook-501.test.ts` — pins that the stub returns 501 with a "Not Implemented" body. If/when v1.1 lands, that test is replaced (not deleted) with the real-handler integration test.
